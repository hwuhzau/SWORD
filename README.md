## 数据格式规范

### 1. 表型数据 (Phenotype Data)

**文件名**: `sample_174.csv`

**格式要求**:
- 第2列（索引为1）必须包含表型数值
- 行数 = 样本/材料数量

**示例**:
```csv
sample_id,phenotype_value
sample1,5.23
sample2,4.87
sample3,6.12
```

---

### 2. 基因列表文件

**文件名**: `gene_test.csv`

**格式要求**:
- 第1列（索引为0）包含待分析的基因ID列表
- 这些基因ID必须在至少一个组学数据中出现

**示例**:
```csv
gene_id
Zm00001d027231
Zm00001d027232
Zm00001d027233
```

---

### 3. SNP数据 (基因组学数据)

**文件名**: `snp_2247428_174_012.csv`

**格式要求**:
- **Shape**: `(SNP数量, 9 + 材料数量)`
- **前9列固定格式**:
  1. `CHROM`: 染色体标识
  2. `POS`: SNP位置标识
  3. **`GENE_ID`**: 基因ID（必需，用于关联基因）
  4. `REF`: 参考位点
  5. `ALT`: 替代位点
  6-9. 其他信息列（可为空）
  
- **材料列**: 第10列开始为各材料的基因型数据
  - 编码: 0, 1, 2（012编码）
  - 缺失值: 用 `-1` 表示

**示例**:
```csv
CHROM,POS,GENE_ID,REF,ALT,name6,name7,name8,name9,ZHENG58,ZHENG653,ZHI41,ZZ01
S1_51053,A/T,Zm00001d027231,51053,+,,,,0,1,0,0
S1_52463,C/T,Zm00001d027231,52463,+,,,,1,1,0,1
```

**重要**: 第3列必须是 `GENE_ID` 信息！

---

### 4. 中间分子组学数据

**适用文件**:
- `rna_174_28279_norm.csv` (转录组数据)
- `ribo_174_18426_norm.csv` (核糖体组数据)
- `pro_174_6210_norm.csv` (蛋白质组数据)

**格式要求**:
- **Shape**: `(材料数量, 基因数量)`
- **行**: 每行代表一个材料/样本
- **列**: 每列代表一个基因，列名为基因ID
- **数值**: 已进行 Z-Score 标准化处理

**示例**:
```csv
Zm00001d027231,Zm00001d027232,Zm00001d000452,Zm00001d000458
0.95566,0.124925,0.161932,0.904985
0.35726,0.098795,0.223663,0.688800
```

**数据预处理要求**:
- 所有数值必须经过 Z-Score 标准化
- 列名必须与基因ID严格对应

---

### 5. 组学数据缺失处理

**如果某个组学数据不存在**，可以设置为空的 DataFrame：

```python
# 示例：如果没有蛋白质组数据
pro_data = pd.DataFrame([])
```

**注意**: 代码最多支持4个组学层（1个基因组学 + 3个中间分子组学）的整合分析。

## 文件说明

### 1. `train_model.py`
**功能**: 主训练脚本，使用真实表型数据训练多组学整合模型

**主要步骤**:
1. 加载所有组学数据和表型数据
2. 对每个基因进行10折交叉验证
3. 整合多组学特征
4. 评估模型性能（R² score）
5. 计算各组学层的贡献度

**输出文件**:
- `.observed_results.csv`: 真实数据的模型结果（每个基因每折的R²和最优子网络）
- `.omics_contributions.csv`: 各组学对每个样本预测值的贡献分解

---

### 2. `permutation_test.py`
**功能**: 置换检验，生成零假设分布

**主要步骤**:
1. 随机打乱表型数据（保持特征不变）
2. 对每个基因执行20次置换 × 10折交叉验证
3. 记录置换后的模型性能

**输出文件**:
- `.permuted_results.csv`: 置换检验的零分布结果（用于统计检验）

**说明**: 此步骤用于评估观测到的模型性能是否显著优于随机期望。

---

### 3. `compute_pvalues.py`
**功能**: 计算统计显著性（p值）

**输出文件**:
- `.uncorrected_pvalues.csv`: 未校正的原始p值

---

### 4. `bonferroni_correction.py`
**功能**: 多重假设检验校正

**主要步骤**:
1. 读取未校正的p值
2. 应用Bonferroni校正（p值 × 基因总数）
3. 重新计算-log10(p)
4. 筛选显著基因（p < 0.001）

**输出文件**:
- `.corrected_pvalues.csv`: Bonferroni校正后的所有基因p值
- **`significant_genes.csv`**: 最终的显著性基因列表（p < 0.001）

## 使用流程

### 步骤1: 准备数据

确保所有数据文件按照上述格式规范准备完毕，并放置在脚本同级目录下：

```
project/
├── 1train_model.py
├── 2permutation_test.py
├── 3compute_pvalues.py
├── 4bonferroni_correction.py
├── sample_174.csv
├── gene_test.csv
├── snp_2247428_174_012.csv
├── rna_174_28279_norm.csv
├── ribo_174_18426_norm.csv
└── pro_174_6210_norm.csv
```

---

### 步骤2: 执行分析

**按顺序执行以下4个脚本：**

```bash
# 1. 训练真实数据模型
python 1train_model.py

# 2. 执行置换检验（耗时较长）
python 2permutation_test.py

# 3. 计算p值
python 3compute_pvalues.py

# 4. 多重检验校正并筛选显著基因
python 4bonferroni_correction.py
```

### 最终结果文件

**`significant_genes.csv`** ⭐

**列说明**:
- `gene_id`: 基因ID
- `p_value`: Bonferroni校正后的p值
- `-log(p_value)`: -log10(p值)
- `voted_subnet`: 最优子网络组合 [SNP, RNA, Ribo, Pro]
  - `1`: 该组学被选入模型
  - `0`: 该组学未被选入模型
  - `-1`: 该组学数据不存在

**示例**:
```csv
gene_id,p_value,-log(p_value),voted_subnet
Zm00001d027231,0.0001,4.0,"[1, 1, 0, 1]"
Zm00001d027232,0.0005,3.3,"[1, 1, 1, 0]"
```

## 注意事项

### ⚠️ 计算性能与可重复性

**默认设置 (并行计算)**:
- 代码使用 `n_jobs=-1` 实现多核并行加速
- **优点**: 速度快，充分利用CPU资源
- **缺点**: 由于并行任务调度的随机性，每次运行结果可能略有差异

**可重复性设置 (串行计算)**:
```python
# 在以下文件中修改：
# 1train_model.py
# 2permutation_test.py

# 将这行：
all_results = Parallel(n_jobs=-1, verbose=1)(...)

# 改为：
all_results = Parallel(n_jobs=1, verbose=1)(...)
```

**修改后**:
结果完全可重复,运行速度变慢
---

### 📊 数据质量检查

运行前建议检查：

1. **表型数据**: 无缺失值，数值合理
2. **基因ID一致性**: `gene_test.csv` 中的基因ID在至少一个组学中存在
3. **SNP数据**: 第3列确实是 `GENE_ID`，编码为0/1/2/-1
4. **组学数据**: 已完成Z-Score标准化
5. **样本数一致**: 所有组学数据的样本数相同

---

### 💾 磁盘空间

中间文件可能占用较大空间：
- `.permuted_results.csv`: 通常最大（数GB）
- 建议预留至少 **10-20GB** 磁盘空间

---

### 🔧 故障排查

**常见问题**:

1. **内存不足 (MemoryError)**
   - 减少并行进程数: `n_jobs=4` 或 `n_jobs=2`
   - 分批处理基因列表

2. **找不到基因数据**
   - 检查基因ID拼写和大小写
   - 确认基因在组学数据列名中存在

3. **p值全为NaN**
   - 检查置换检验是否正确运行
   - 确认 `.permuted_results.csv` 文件存在且完整
